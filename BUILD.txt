射频计算器（目前仅pa）— 打包说明（精简）
==========================================

■ 发布目录 dist\（v1.0.0：exe、静态网页、apk 同夹）
  - RF-Calculator-PA-<version>-x64-portable.exe   Windows x64 便携版
  - web-v3\                         静态网页（整份 www 拷贝）
  - RF-Calculator-PA-v1.0.0-arm64-debug.apk   Android ARM64 调试包（与 build-apk-arm64.cmd 中文件名一致）
  - RF-Calculator-PA-arm64-debug.apk      与上一行同一 APK 的副本
  build-win.cmd 会删除 dist\win-unpacked，再写入便携 exe 与 web-v3\。

■ 对外发布（GitHub Releases）
  用户下载入口以 Releases 为准。同步 docs 后，可生成 artifacts\RF-Calculator-PA-v1.0.0-web-offline.zip（与 README 中附件名一致），
  再与 exe、apk 一并上传。自动上传：设置环境变量 GITHUB_TOKEN 后执行
    powershell -ExecutionPolicy Bypass -File scripts\publish-github-release.ps1
  大体积 exe/apk 勿提交到 Git（见根目录 .gitignore）。

■ 国内网络（脚本 + 工程内固定源）
  1) 项目根 .npmrc：registry=https://registry.npmmirror.com（npm install 默认走国内，不依赖先跑 env-china）
  2) scripts\env-china.cmd（由 build-*.cmd 自动调用）额外设置：
     NPM_CONFIG_REGISTRY、ELECTRON_MIRROR、ELECTRON_BUILDER_BINARIES_MIRROR、CSC_IDENTITY_AUTO_DISCOVERY=false
  3) npm run android:patch 即 scripts\patch-android.cjs（cap sync 后执行），会写入：
     - android\gradle\wrapper\gradle-wrapper.properties
       → 腾讯云 Gradle 包；networkTimeout=120000；validateDistributionUrl=false
     - android\settings.gradle
       → 文件首段 pluginManagement.repositories 仅阿里云（解析 Android Gradle 插件等）
     - android\build.gradle
       → buildscript / allprojects 前置阿里云，并移除 google()、mavenCentral() 直连（依赖经镜像拉取）
     - variables.gradle、app abiFilters arm64-v8a
  Android SDK 本体仍由 Android Studio / sdkmanager 从官方或你自配的镜像安装；本仓库只保证 Gradle/Maven/npm/Electron 侧走国内源。
  若需改镜像：可编辑 env-china.cmd、.npmrc 或 patch-android.cjs。

■ 若未生成 APK（常见原因）
  - 未装 JDK 17/21 或未设 JAVA_HOME（仅用 Java 8 会失败）
  - 未装 Android SDK，或未执行 sdkmanager --licenses 接受许可
  - 环境变量 ANDROID_HOME / ANDROID_SDK_ROOT 未指向有效 SDK
  - Gradle 报错：把 android 目录下完整构建日志保留，对照上文逐项排查

■ 工程依赖
  首次或更新依赖: npm install（会走 npmmirror）

■ Windows — 仅 x64 便携 exe + 网页目录
  build-win.cmd
  输出: dist\RF-Calculator-PA-*-portable.exe 与 dist\web-v3\
  productName 为 ASCII（RF-Calculator-PA）；中文窗口标题见 electron-main.cjs 的 APP_TITLE_ZH。
  package.json 中 win.signAndEditExecutable=false，避免 winCodeSign 从 GitHub 下载失败。

■ Android — 仅 arm64-v8a 调试 APK
  必须: JDK 17 或 21（AGP 8.2 要求运行 Gradle 的 JVM 为 Java 11+；仅用 Java 8 会报 “compatible with Java 8” 且无法解析 classpath）。
        推荐: winget install Microsoft.OpenJDK.21
        若已安装 Microsoft OpenJDK 21 但未设 JAVA_HOME，build-apk-arm64.cmd 会自动使用
        C:\Program Files\Microsoft\jdk-21*\ ；否则请自行 set JAVA_HOME 并把 %%JAVA_HOME%%\bin 加入 PATH。
  另需: Android SDK、sdkmanager --licenses
  build-apk-arm64.cmd
  首次无 android\ 会自动 cap add android；随后 cap sync 与 android:patch。
  Gradle 产物: android\app\build\outputs\apk\debug\app-debug.apk
  发布副本: dist\RF-Calculator-PA-v1.0.0-arm64-debug.apk（及 RF-Calculator-PA-arm64-debug.apk）
  若升级 package.json / android versionName，请同步修改 build-apk-arm64.cmd 中带版本号的复制目标文件名。

■ 一次打 exe + APK + 网页（国内镜像）
  build-all.cmd

■ 离线 Chart.js
  www\vendor\chart.umd.min.js
  更新: scripts\refresh-vendor-chart.cmd（默认仍从 jsDelivr 拉取；国内可自备文件覆盖）

■ 浏览器预览
  start-server.cmd → http://localhost:8080
